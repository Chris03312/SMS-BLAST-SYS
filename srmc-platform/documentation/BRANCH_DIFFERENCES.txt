 5. BRANCH: v3
--------------------------------------------------------------------------------

  Latest commit: 5cc8529 - "full integration of internet based"
  Unique commits (not on master or v2):
    5cc8529  full integration of internet based
    79736f0  fix: syntax error - wrap ?? with || in parentheses
    ec67190  PULL mode: phone_id matching, mode selector, Lite device ID
    7b4ea05  Epal na DB
    3896253  lite app mini refactoring
    67dcdbe  db update  (also on master)

  Description:
    v3 is the most advanced branch and the only one with the "PULL mode"
    architecture. This is a fundamentally different approach to SMS delivery
    where the Android gateway phones actively PULL pending messages from the
    server (instead of the server PUSHing to the phones).

    KEY CHANGES vs master:
      • PULL mode: phone_id matching, mode selector, Lite device ID
      • Full integration of "internet-based" delivery
      • SRMCGatewayLite app support (a lighter Android gateway APK)
      • Major broadcast-engine refactor (622 insertions, 256 deletions)
      • Removal of broadcast-helpers.js and sim-utils.js
      • Significant Analytics page overhaul
      • Major gateway-service.js and gateway-poller.js restructuring

  The "full integration of internet based" commit (5cc8529) is a small
  finishing commit that caps the PULL mode work:
  • 10 files changed, only 93 insertions
  • Main changes in gateway-poller.js (40 lines), broadcasts.js (20),
    MainActivity.java (51), and Gateway.jsx (14)
  • This is the final touch on top of the PULL mode infrastructure built
    in the preceding commits (ec67190, 3896253, 7b4ea05)

  💡 NOTE on "internet based" across branches:
     Both master and v3 have commits mentioning "internet based", but they
     are DIFFERENT commits with DIFFERENT approaches:
     - master: big monolithic commit that ADDS broadcast-helpers + sim-utils
     - v3: small finishing commit on top of PULL mode — v3 actually
           REMOVES broadcast-helpers.js and sim-utils.js because the PULL
           mode architecture made them unnecessary

    FILES REMOVED in v3 (compared to v2):
      • packages/client/src/pages/admin/Contacts.jsx
      • packages/client/src/pages/agent/Recipients.jsx
      • packages/server/routes/contacts.js
      • packages/server/services/broadcast-helpers.js
      • packages/server/services/sim-utils.js
      • documentation/HOW_PULL_MODE_WORKS.txt

    v3 does NOT include v2's recipients/contacts features. The recipients
    management was apparently replaced by the PULL mode architecture where
    devices are identified by phone_id rather than by manual assignment.

  Diff stats vs v2:  41 files changed, 1,164 insertions, 2,876 deletions

 7. RECOMMENDATION
--------------------------------------------------------------------------------

  • Use master if you want the current stable release with internet-based
    sending capability and general enhancements.

  • Use v2 if you need the recipients/contacts management features (upload
    via CSV, auto-select, boss number per campaign). Note that v2 does NOT
    have the latest internet-based improvements from master.

  • Use v3 if you need the full PULL mode architecture where Android devices
    autonomously pull messages from the server. This is the most advanced
    branch with a fundamentally reworked delivery engine and Lite app
    support, but it sacrifices the v2 recipients management features.

  💡 Consider merging features across branches (e.g., cherry-pick v2's
     recipients/contacts into v3) if you need both recipient management AND
     the PULL mode architecture.
